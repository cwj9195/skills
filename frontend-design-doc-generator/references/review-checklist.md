# Frontend Spec Review Checklist

> Review Agent 在交付 `requirements.md / facts.md / design.md / tasks.md` 前必须执行本清单。

## 一、产物完整性

| # | 检查项 | 通过标准 | 状态 | 说明 |
| - | ------ | -------- | ---- | ---- |
| 1 | requirements.md | 覆盖目标、范围、页面、流程、字段、权限、验收 | ✅/❌ | TODO |
| 2 | facts.md | 包含代码库、组件、页面、请求、规范 Evidence | ✅/❌ | TODO |
| 3 | design.md | 使用 frontend-design-doc-generator 模板 | ✅/❌ | TODO |
| 4 | tasks.md | 可交给编码 Agent 分任务执行 | ✅/❌ | TODO |

## 二、Evidence 门禁

| # | 检查项 | 通过标准 | 状态 | 说明 |
| - | ------ | -------- | ---- | ---- |
| 1 | Codegraph 优先 | 结构、符号、影响面已先用 Codegraph | ✅/❌ | TODO |
| 2 | 代码事实 | 关键路径、组件、请求、路由有 Evidence | ✅/❌ | TODO |
| 3 | PRD 事实 | 需求结论可回溯到 PRD 或 TODO | ✅/❌ | TODO |
| 4 | 原型事实 | MasterGo/Figma 信息有 DSL/截图/链接证据或 TODO | ✅/❌ | TODO |
| 5 | 未编造 | 无伪造路径、组件、接口、权限码 | ✅/❌ | TODO |

## 三、设计书质量

| # | 检查项 | 通过标准 | 状态 | 说明 |
| - | ------ | -------- | ---- | ---- |
| 1 | 章节完整 | Implementation Mode 保留 13 章节 | ✅/❌ | TODO |
| 2 | 表格完整 | 字段字典、API、状态机、权限、TODO 表格列完整 | ✅/❌ | TODO |
| 3 | 组件映射 | 组件选择优先使用企业组件映射 | ✅/❌ | TODO |
| 4 | 数据链路 | 查询、保存、刷新、回填、错误处理明确 | ✅/❌ | TODO |
| 5 | 状态机 | 关键状态和禁用动作明确 | ✅/❌ | TODO |
| 6 | 验收标准 | 使用 Given/When/Then，不写“功能正常” | ✅/❌ | TODO |

## 四、任务可执行性

| # | 检查项 | 通过标准 | 状态 | 说明 |
| - | ------ | -------- | ---- | ---- |
| 1 | 任务边界 | 每个任务可独立执行或依赖清晰 | ✅/❌ | TODO |
| 2 | 文件范围 | 每个任务列出涉及文件或待确认路径 | ✅/❌ | TODO |
| 3 | 完成标准 | 每个任务有明确完成标准 | ✅/❌ | TODO |
| 4 | 回归检查 | 包含点击、回调、权限、保存、失败反馈 | ✅/❌ | TODO |

## 五、最终结论

| 结论 | 条件 | 处理 |
| ---- | ---- | ---- |
| 可交给编码 Agent | 所有 P0/P1 检查通过，非阻塞 TODO 有临时方案 | 进入实现 |
| 阻塞，不可进入编码 | 缺 PRD 核心范围、关键接口、关键组件 Evidence 或权限规则 | 补充输入后重审 |
| 可先做 Outline | PRD 范围明确但代码/API/原型 Evidence 不足 | 输出提纲版，不进入编码 |

# 企业组件映射规则

> 用于 Frontend Spec Agent 和 frontend-design-doc-generator。组件选择必须先查本文件，再结合 Codegraph Evidence 和相邻页面示例确认。

## 使用原则

- 优先复用企业组件和当前项目已有模式。
- 未找到真实组件路径或示例时，不编造；写 `(待确认)` 或 TODO。
- 如果组件映射与代码 Evidence 冲突，以代码 Evidence 为准，并把冲突写入 TODO。
- 设计书中必须说明组件职责、关键 props、事件、插槽、复用示例和 Evidence。

## 默认映射表

| 业务语义 | 优先组件/模式 | 禁止或慎用 | 需要补充的 Evidence |
| -------- | ------------- | ---------- | ------------------- |
| 筛选表单 | `FormPro` / `FormItemPro` | 无证据直接使用裸 `a-form` | 相邻列表页筛选区 |
| 编辑表单 | `FormPro` / `FormItemPro` | 将复杂表单拆成散落控件 | 表单提交、校验、回填示例 |
| 字典选择 | `DictPro` | 手写枚举且无字典来源 | 字典 key、展示组件、回显规则 |
| 状态展示 | `DictPro` / Tag 模式 | 纯文本状态且无说明 | 状态码、文案、颜色规则 |
| 权限按钮 | `Permission` | 只用 `disabled` 替代权限 | 权限码、隐藏/禁用规则 |
| 列表表格 | `vxe-table` / 项目 TablePro 模式 | 无 Evidence 新增表格方案 | 分页、列配置、行操作示例 |
| 详情抽屉 | `useDrawer` / 项目现有抽屉模式 | 临时弹窗方案 | 打开、关闭、刷新、回填链路 |
| 弹窗确认 | 项目现有 Modal/Confirm 模式 | 无确认态直接提交 | 二次确认、loading、失败反馈 |
| 图片上传 | `tc-upload` | 直接裸上传控件 | 文件限制、回显、删除、错误处理 |
| 图片裁剪 | `tc-cropper` | 新增裁剪实现 | 裁剪尺寸、上传联动示例 |
| 类目选择 | `material-category-selector` | 手写类目树 | 类目层级、回显、清空规则 |
| 供应商选择 | `SelectSearch` / 项目现有远程选择模式 `(待确认)` | 静态 select | 查询接口、分页、回显字段 |
| 导入 | 项目上传/导入模式 `(待确认)` | 单纯文件上传无结果处理 | 模板下载、上传、校验结果 |
| 导出 | 项目导出模式 `(待确认)` | 直接拼 URL | 参数、权限、loading、失败反馈 |

## 输出要求

在 `design.md` 的组件章节中至少说明：

- 组件名和来源路径。
- 现有示例页面。
- 关键 props、emits、slots 或 exposed。
- 使用场景。
- 与 PRD 或 MasterGo 的对应关系。
- 不确定项和 TODO。

## 缺失映射处理

当需求出现本文件未覆盖的业务语义：

1. 使用 Codegraph 搜索相邻页面或相似组件。
2. 若找到 Evidence，补充到 `facts.md` 并在设计书中引用。
3. 若未找到，写入 TODO，说明推荐默认方案和风险。
4. 后续沉淀时再补回本文件。
